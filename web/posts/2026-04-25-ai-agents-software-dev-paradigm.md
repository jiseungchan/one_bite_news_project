## 기존에는 어땠나

지난 2년간 개발자 생산성 도구의 패러다임은 'AI 자동완성'이었다. GitHub Copilot이 2021년에 등장하면서 IDE(통합개발환경) 안에서 한 줄씩 코드를 제안하는 방식이 표준이 됐다. 개발자는 여전히 운전대를 잡고, AI는 조수석에서 지도를 보는 구조였다.

이 구조의 한계는 분명했다. AI가 제안한 코드가 전체 프로젝트 맥락에서 작동하는지 검증하는 작업은 여전히 사람의 몫이었다. 자동완성이 빠를수록 오히려 검증에 드는 시간이 늘었다는 아이러니도 있었다. 실제로 2025년 McKinsey 조사에서 개발자들은 "AI 도입 이후 코딩 속도는 40% 올랐지만, 코드 리뷰 시간이 60% 늘었다"고 답했다.

## 무엇이 다른가

지금 일어나는 변화는 단순한 속도 개선이 아니다. AI 에이전트(agent — 스스로 목표를 세우고 행동을 연속적으로 실행하는 AI 시스템)가 소프트웨어 개발의 전 주기를 처음부터 끝까지 담당하기 시작했다.

Cognition AI가 공개한 에이전트 시스템 'Devin'은 자연어로 작성된 요구사항을 입력받아 스스로 코드를 작성하고, 에러를 수정하며, 테스트 케이스를 실행한 뒤 풀 리퀘스트(pull request — 코드 변경 제안서)까지 올린다. GitHub도 2026년 초 'Copilot Workspace'를 정식 출시하면서 이슈(버그 리포트)를 에이전트가 직접 분석해 수정 코드를 제안하는 흐름을 도입했다.

변화의 핵심은 **자율성**이다. 기존 자동완성은 인간의 입력에 반응하는 수동적 도구였다면, 에이전트는 목표만 주어지면 도구를 선택하고 계획을 세워 실행한다. 이를 'agentic loop(에이전틱 루프)'라고 부른다 — 계획 → 실행 → 검증 → 수정을 사람의 개입 없이 반복하는 구조다.

| 구분 | 기존 AI 코딩 도구 | AI 에이전트 |
|------|-----------------|------------|
| 작업 단위 | 한 줄, 한 함수 | 전체 태스크·프로젝트 |
| 인간 개입 | 매 단계 | 시작과 검수 시점만 |
| 도구 활용 | 없음 | 터미널·브라우저·API 직접 호출 |
| 에러 처리 | 제안만 함 | 스스로 디버깅·재시도 |

## 어디에 쓰이나

이미 실제 현장에서 변화가 감지된다. Cognition AI에 따르면 Devin을 도입한 기업들은 단순 버그 수정 작업의 60~70%를 에이전트에게 위임했다. Microsoft는 Azure DevOps 파이프라인에 에이전트 기능을 통합해 CI/CD(지속적 통합·배포) 과정에서 발생하는 테스트 실패를 자동으로 분석·수정하는 기능을 베타 서비스 중이다.

영향을 받는 직군 범위도 넓어지고 있다. QA(품질보증) 엔지니어, 주니어 개발자, 기술 문서 작성자가 담당하던 반복성 높은 업무가 에이전트로 이전되는 속도가 빠르다. 반면 시스템 아키텍처 설계, 요구사항 협상, 에이전트 감독 등 판단이 필요한 역할의 수요는 높아졌다.

## 병규의 한 줄

이번 흐름에서 흥미로운 건 도구가 아니라 책임 구조의 이동이다. AI가 코드를 짜는 게 아니라 '개발 태스크의 소유자'가 되는 순간, 인간 개발자의 역할은 실행자에서 검수자·설계자로 급격히 이동한다. 소프트웨어 개발이라는 직업의 정의 자체가 바뀌고 있다.
