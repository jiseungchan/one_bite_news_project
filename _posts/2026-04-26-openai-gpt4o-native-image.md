## 배경 — 이미지 생성은 늘 '외주'였다

지금까지 ChatGPT에서 이미지를 만들려면 GPT 모델이 DALL-E(달리)라는 별도 이미지 생성 모델을 호출하는 방식이었다. 텍스트를 이해하는 모델과 그림을 그리는 모델이 분리된 구조다. 이 방식은 단순해 보이지만 실제로는 여러 한계가 있었다. 텍스트 맥락이 이미지로 넘어가는 과정에서 정보가 손실되고, 대화 흐름 속에서 앞선 맥락을 반영한 이미지를 만들기가 어려웠다. "아까 말한 그 캐릭터를 이렇게 바꿔줘" 같은 요청이 자연스럽게 이어지지 않는 이유가 바로 이 구조적 분리 때문이었다.

업계에서는 이 문제를 해결하기 위해 '멀티모달 통합'을 오랫동안 목표로 삼아왔다. 텍스트, 이미지, 오디오를 하나의 모델이 동시에 다루는 구조다. OpenAI가 GPT-4o를 발표하면서 이 방향을 선언했지만, 초기에는 이미지 생성만큼은 여전히 DALL-E에 의존하는 형태였다.

## 핵심 — 하나의 모델이 텍스트와 이미지를 함께 처리

이번 업데이트의 핵심은 GPT-4o가 외부 이미지 생성 모델을 호출하지 않고, 모델 자체에서 바로 이미지를 출력한다는 점이다. 기술적으로는 '오토레그레시브(autoregressive) 이미지 토큰 생성' 방식을 사용한다. 텍스트 토큰을 순서대로 생성하듯, 이미지 픽셀을 토큰 형태로 순차 생성하는 구조다.

이게 실제로 어떻게 다른가:

| 항목 | 기존 방식 (DALL-E 외주) | 네이티브 통합 방식 |
|------|------------------------|-------------------|
| 모델 수 | 2개 (언어 + 이미지) | 1개 |
| 대화 맥락 유지 | 부분적 | 완전 |
| 이미지 편집 연속성 | 제한적 | 자연스럽게 이어짐 |
| 텍스트+이미지 혼합 출력 | 불가 | 가능 |

가장 실질적인 변화는 대화 맥락 유지다. 앞선 대화에서 설명한 설정이나 스타일을 그대로 이어받아 이미지를 생성하거나 수정할 수 있다. 기존에는 이미지 생성 요청이 새 맥락에서 시작되는 느낌이었다면, 이제는 대화 흐름 속에서 자연스럽게 그림이 나오는 구조다.

또 하나 주목할 부분은 텍스트와 이미지가 섞인 출력이다. 예를 들어 다이어그램 설명과 실제 다이어그램 이미지를 함께 생성하거나, 이미지 안에 정확한 텍스트를 삽입하는 작업이 가능해졌다. 기존 DALL-E가 이미지 내 텍스트 렌더링에 약했던 것과 대비된다.

## 시장 영향 — 크리에이티브 툴 시장에 직접 충격

- ChatGPT 유료 구독자(Plus, Pro, Team) 대상 우선 배포, 무료 사용자 순차 확대 예정
- 어도비 파이어플라이(Firefly), 미드저니(Midjourney), 스태빌리티AI(Stability AI) 등 전문 이미지 생성 서비스와 직접 경쟁 구도 형성
- 콘텐츠 제작 워크플로우에서 별도 이미지 생성 툴이 필요 없어지는 방향으로 수렴 중
- 개발자 API를 통한 통합도 지원, B2B 이미지 자동화 솔루션 시장에도 영향

크리에이티브 도구 시장에서 가장 민감하게 반응하고 있는 건 어도비다. 포토샵·일러스트레이터 생태계를 AI로 보강하는 전략을 쓰고 있는 어도비 입장에서, ChatGPT가 이미지 생성까지 원스톱으로 처리하게 되면 구독 이탈 가능성이 생긴다.

## 병규의 한 줄

이번에 흥미로운 건 기능보다 구조다. '텍스트 모델 + 이미지 모델'이라는 조합 방식을 버리고 하나의 모델에서 모든 모달리티를 처리하는 방향이 AI 업계의 다음 표준이 될 것임을 OpenAI가 확실히 선언했다. 이 구조 변화가 본격화되면, 모달리티별 전문 모델이라는 개념 자체가 흐려진다.
