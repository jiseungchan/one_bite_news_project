## 한 줄 요약

> 소형 모델이 실행을 맡고 Opus가 분기점에서만 조언하는 구조로, 성능과 비용을 동시에 개선했다.

---

## 리드 — 왜 지금 이게 중요한가?

Haiku 4.5에 Opus 4.6 어드바이저를 붙였더니 BrowseComp 점수가 19.7%에서 41.2%로 뛰었다. 두 배 이상이다. 비용은 Sonnet 단독 대비 85% 저렴하다. 앤트로픽이 2026년 4월 9일 공개한 '어드바이저 전략'이 만들어낸 숫자다.

---

## 배경 — 기존에는 어땠나?

지금까지 멀티에이전트 구조의 기본 상식은 하나였다. 대형 모델이 지휘하고, 소형 모델이 실행한다. 오케스트레이터 자리는 항상 가장 똑똑한 모델의 몫이었다.

문제는 AI 에이전트 작업 대부분이 사실 그렇게 복잡하지 않다는 점이다. 도구 호출, 파일 읽기, 오류 재시도 같은 반복 처리는 소형 모델로도 충분하다. 그럼에도 대형 모델을 상시 투입하면 비용이 올라가고, 에이전트가 복잡해질수록 이 비효율이 눈덩이처럼 커진다. 가트너에 따르면 에이전트 작업은 기존 챗봇 대비 작업당 5~30배 더 많은 토큰을 소비한다. (출처: 인공지능신문 / 가트너)

---

## 핵심 기술 — 무엇이 다른가?

### 원리

어드바이저 전략은 역할 구조를 뒤집는다. 소형 실행자 모델(Sonnet 4.6 또는 Haiku 4.5)이 작업 전체를 주도한다. Opus 4.6은 실제로 추론이 필요한 분기점 — 파일 수정 대상 결정, 리팩터링 여부 판단, 작업 분해 전략 수립 등 — 에서만 호출된다.

Opus가 조언할 때 내놓는 건 400~700 토큰짜리 짧은 계획이 전부다. 긴 출력은 모두 소형 모델 몫이다. `max_uses` 파라미터로 Opus 호출 횟수에 상한선을 설정할 수 있어 비용 예측도 가능하다.

### 기존 대비 차별점

| 항목 | 기존 Top-Down 방식 | 어드바이저 전략 |
|------|------------------|----------------|
| 오케스트레이터 | 대형 모델 (항상 개입) | 소형 모델 (전체 주도) |
| 대형 모델 역할 | 지휘·위임 | 분기점 조언만 |
| 대형 모델 출력량 | 전체 응답 생성 | 1회 400~700 토큰 |
| 비용 구조 | 대형 모델 비중 높음 | 소형 모델 위주, 예측 가능 |

## 시장 영향 — 어디에 쓰이나?

성능 수치를 두 가지로 정리하면 이렇다.

- **Sonnet 4.6 + Opus 조합**: SWE-벤치 다국어 72.1% → 74.8% (+2.7%p), 비용 11.9% 절감
- **Haiku 4.5 + Opus 조합**: BrowseComp 19.7% → 41.2% (2배 이상 향상), 비용 85% 절감

어드바이저 전략은 앤트로픽이 4월 8일 함께 출시한 Claude Managed Agents 플랫폼 위에서 작동한다. 노션, 라쿠텐, 아사나, 센트리, 바이브코드가 이미 도입했다.

가트너도 같은 방향을 권고하고 있다 — "반복·고빈도 업무는 소형 모델, 복잡한 추론은 대형 모델에 엄격히 한정." 앤트로픽은 이를 API 수준에서 구현한 셈이다.

## 병규의 한 줄

> 이번에 흥미로운 건 수치 자체보다 설계 철학이다. "대형 모델이 항상 더 많이 관여할수록 좋다"는 가정을 깨고, 개입을 최소화하는 쪽으로 성능을 끌어올렸다는 점.
